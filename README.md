# Finding Your Neighborhood (FYN)

A static site generator for intentional community doors.

FYN lets you create a simple, creator-owned website that describes a space and points people toward where connection already happens. It does not track visitors, collect data, infer relationships, or maintain a directory. The creator defines the links. That's the whole mechanism.

---

## What it does

You launch a lightweight local tool, fill out a web form, and FYN generates a folder of static HTML files ready to publish to GitHub Pages. The result is a small, clean site with:

- A home page describing the space and who it's for
- A links page pointing to real places people should go next
- No backend, no database, no third-party scripts

---

## Why it exists

Integrity is enforced by the build process, not by policy. That's the design principle FYN is built around.

Most connection tools optimize for visibility, engagement, or growth. FYN does none of these things. It assumes that communities already exist and that the problem is findability, not building something new. A tiger conservationist in Minnesota and one in Manitoba may share vocabulary, tools, and working knowledge without knowing each other exists. A FYN door built around that specific language is findable by the people who belong there. If the generated output contains tracking patterns, analytics scripts, or hidden data collection, the build fails. Nothing ships.

## Requirements

- Python 3.x
- Git (for publishing to GitHub Pages)
- A GitHub repository configured for GitHub Pages

---

## Getting started

1. Clone or download this repository.
2. Run `python creator_server.py` from the project folder, or double-click `start.bat` on Windows.
3. A web form opens in your browser.
4. Fill in your space name, description, and two or more places people should go.
5. Click **Create Site**. FYN writes `config.json`, builds the `docs/` folder, and opens a preview.
6. Publish manually: `git add docs`, `git commit -m "Update site"`, `git push`.

## Project structure

```
fyn/
├── start.bat               # Launch script (Windows)
├── creator_server.py       # Local web server and form handler
├── build.py                # Site builder and integrity checker
├── config.json             # Generated by the form; do not hand-edit
├── CNAME                   # Optional custom domain; copied into docs/ if present
├── templates/
│   ├── base.html           # Page shell
│   └── page.html           # Page content block
├── assets/
│   └── style.css           # Styles
└── docs/                   # Generated output — publish this folder
```

## Integrity constraints

The build will hard-stop if the generated output contains:

- Analytics scripts (Google Analytics, Google Tag Manager, etc.)
- Advertising or tracking pixels
- Cookie access (`document.cookie`)
- Local or session storage access
- Background network requests (`fetch`, `XMLHttpRequest`, `sendBeacon`)
- Third-party tracking domains

This is not a warning. The build fails and nothing is written.

---

## Current limitations

- The form creates only a Home page and a Links page. Additional page types are planned.
- Publishing to GitHub requires running git commands manually. An integrated push option is planned.
- The form HTML is embedded in `creator_server.py`. This works but is not ideal for long-term maintenance.
- The integrity check uses string pattern matching. Stronger validation is planned.

---

## Editing an existing site

Edit `config.json` directly, then run `python build.py` to rebuild. The form does not currently reload an existing config for editing — that is a known limitation planned for a future version. If you're new to Git, GitHub's documentation covers the push process.

---

## Configuration reference

`config.json` is the source of truth for the site. A minimal example:

```json
{
  "site": {
    "title": "Space Name",
    "description": "One sentence describing who this is for."
  },
  "pages": [
    {
      "slug": "index",
      "title": "Home",
      "intro": "One sentence intro.",
      "body": ["Short paragraph describing the space."]
    },
    {
      "slug": "links",
      "title": "Where People Should Go",
      "intro": "Creator-defined places and resources.",
      "body": ["These places are chosen by the creator."],
      "link_groups": [
        {
          "title": "Places to go next",
          "links": [
            {
              "label": "Place name",
              "url": "https://example.com",
              "description": "Why someone would go here.",
              "type": "Place to go"
            }
          ]
        }
      ]
    }
  ]
}
```

---

## Custom domain

Add a `CNAME` file to the project root containing your domain. It will be copied into `docs/` automatically on each build.
